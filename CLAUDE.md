# CLAUDE.md — правила работы над проектом HA-Notifications

## О проекте

Кросс-платформенный агент `homecrier` на Go: подключается к WebSocket API Home Assistant и показывает нативные системные уведомления на Windows 11 и macOS при смене состояния датчиков. Полная архитектура — в `docs/architecture.md`, спецификация — в `docs/spec.md`.

## Обязательные правила

- Общение и комментарии в коде — **строго на русском языке**.
- В начале каждого файла/модуля/пакета — комментарий с описанием его логики.
- Перед выполнением любой задачи из `docs/tasks/` прочитать `docs/spec.md` — спецификация первична; при противоречии задача уточняется, а не спецификация молча нарушается.
- Секреты (токен HA) никогда не коммитятся: только переменная окружения `HA_TOKEN` или файл, указанный в `.gitignore`.

## Технологический стек (зафиксирован в ADR, не менять без нового ADR)

- Go ≥ 1.22, модуль `homecrier`.
- WebSocket: `github.com/coder/websocket`.
- Уведомления: `github.com/gen2brain/beeep` (базовый уровень для обеих ОС).
- Конфиг: YAML (`gopkg.in/yaml.v3`).
- Логирование: стандартный `log/slog`.

## Структура репозитория

```
docs/           — архитектура, ADR, спецификация, инструкции, задачи
cmd/agent/      — main-пакет агента (создаётся в task-01)
internal/       — пакеты config, hass, notify (создаются по задачам)
scripts/        — сборка и установка (task-06)
```

## Команды

- Сборка: `go build ./...` из корня репозитория.
- Тесты: `go test ./...`.
- Кросс-компиляция — см. `docs/tasks/task-06-build-release.md`.

## Порядок реализации

Задачи в `docs/tasks/` пронумерованы и зависят друг от друга последовательно: task-01 → task-06. Каждая задача считается выполненной только при прохождении её раздела «Критерии приёмки».
