# CLAUDE.md — правила работы над проектом HA-Notifications

## О проекте

Кросс-платформенный агент `homeping` на Go: подключается к WebSocket API Home Assistant и показывает нативные системные уведомления на Windows 11 и macOS при смене состояния датчиков. Полная архитектура — в `docs/architecture.md`, спецификация — в `docs/spec.md`.

## Обязательные правила

- Общение и комментарии в коде — **строго на русском языке**.
- В начале каждого файла/модуля/пакета — комментарий с описанием его логики.
- Перед выполнением любой задачи из `docs/tasks/` прочитать `docs/spec.md` — спецификация первична; при противоречии задача уточняется, а не спецификация молча нарушается.
- Секреты (токен HA) никогда не коммитятся: системное хранилище учётных данных (keyring), переменная окружения `HA_TOKEN` или файл, указанный в `.gitignore`.

## Технологический стек (зафиксирован в ADR, не менять без нового ADR)

- Go ≥ 1.22, модуль `homeping`.
- WebSocket: `github.com/coder/websocket`.
- Уведомления: `github.com/gen2brain/beeep` (базовый уровень для обеих ОС).
- Конфиг: YAML (`gopkg.in/yaml.v3`).
- Логирование: стандартный `log/slog`.
- Трей (v2): `fyne.io/systray` (ADR-003).
- Настройки (v2): встроенный локальный веб-UI — stdlib `net/http` + `go:embed`, без фреймворков (ADR-003).
- Токен (v2): `github.com/zalando/go-keyring` (ADR-003).

## Структура репозитория

```
docs/           — архитектура, ADR, спецификация, инструкции, задачи
cmd/agent/      — main-пакет агента
internal/       — пакеты config, hass, notify; v2 добавляет agent, tray, webui, secrets
scripts/        — сборка и установка
```

## Команды

- Сборка: `go build ./...` из корня репозитория.
- Тесты: `go test ./...`.
- Сборка релиза: Windows — `scripts/build.ps1`; darwin с v2 собирается только на macOS или в CI (cgo из-за systray) — см. `docs/tasks/task-10-packaging.md`.

## Порядок реализации

Задачи в `docs/tasks/` пронумерованы и зависят друг от друга последовательно: v1 — task-01 → task-06 (выполнены), v2 — task-07 → task-10 (реализованы; открыт сквозной сценарий приёмки v2 на реальном окружении, включая MacBook). Каждая задача считается выполненной только при прохождении её раздела «Критерии приёмки».
