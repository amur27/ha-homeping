# ADR-002: Язык и библиотеки агента — Go

- **Статус**: принято
- **Дата**: 2026-07-02

## Контекст

Агент должен работать в фоне на Windows 11 и macOS (Apple Silicon), держать WebSocket-соединение и показывать нативные уведомления. Важны: простота установки (в идеале «скопировал файл — работает»), одна кодовая база, низкое потребление ресурсов, надёжная фоновая работа месяцами.

## Рассмотренные варианты

### 1. Go (выбран)

- ✅ **Один статический бинарник** ~5–10 МБ без зависимостей: не нужен интерпретатор, venv, node_modules на целевых машинах.
- ✅ Кросс-компиляция из одной команды: `GOOS=windows GOARCH=amd64` и `GOOS=darwin GOARCH=arm64`.
- ✅ Горутины + каналы идеально ложатся на задачу «держи соединение, реагируй на события».
- ✅ Потребление памяти единицы мегабайт; для процесса, живущего месяцами, важно отсутствие GC-сюрпризов интерпретаторов.
- ✅ У пользователя уже есть опыт Go-проектов (каталог GoProjects в рабочем окружении).

### 2. Python

- ✅ Быстрый прототип, отличные библиотеки (`hass-client`, `websockets`).
- ❌ На обеих машинах нужен Python + venv + зависимости; обновления Python ломают venv.
- ❌ Нативные уведомления Windows из Python (winrt/win10toast) исторически хрупкие.
- ❌ Автозапуск фонового скрипта с venv через Task Scheduler/launchd заметно капризнее, чем запуск одного exe.

### 3. Node.js / Electron

- ❌ Electron: сотни МБ и целый Chromium ради тоста — несоразмерно задаче.
- ❌ Node без Electron: тот же вопрос рантайма, что и с Python.

### 4. Rust

- ✅ Тоже один бинарник, минимальные ресурсы.
- ❌ Дольше в разработке; экосистема уведомлений/WebSocket не даёт преимуществ над Go для этой задачи.
- Вердикт: равноценный результат за большую цену.

## Решение

Go ≥ 1.22. Библиотеки:

| Назначение | Библиотека | Почему |
|---|---|---|
| WebSocket | `github.com/coder/websocket` | Минималистичная, поддерживает context, активно сопровождается (бывшая nhooyr.io/websocket) |
| Уведомления | `github.com/gen2brain/beeep` | Единый API: на Windows — toast, на macOS — osascript/Notification Center; без cgo |
| Конфиг | `gopkg.in/yaml.v3` | Стандарт де-факто для YAML в Go |
| Логи | `log/slog` (stdlib) | Структурные логи без внешних зависимостей |

## Последствия

- Сборка релиза — простой скрипт кросс-компиляции (task-06).
- Ограничение beeep: без кнопок-действий в уведомлениях. Для v1 достаточно; если понадобятся действия — точечная замена в пакете `internal/notify` (Windows: `go-toast`, macOS: `terminal-notifier`), интерфейс пакета менять не придётся.
- На macOS первое уведомление может потребовать разрешения в System Settings → Notifications (описано в setup-clients.md).
