# ADR-003: UI-стек v2 — systray + локальный веб-интерфейс + системное хранилище токена

- **Статус**: принято
- **Дата**: 2026-07-03

## Контекст

Ядро агента (task-01…06) работает; v2 добавляет пользовательский интерфейс:

1. **Иконка в трее** (Windows) / строке меню (macOS): статус подключения, меню управления.
2. **Настройки в удобной форме** вместо ручного редактирования YAML.
3. Главное ограничение сохраняется: **лёгкость** — один бинарник ~10 МБ, без внешних рантаймов, память в устойчивом состоянии в десятки мегабайт, фоновая работа месяцами.

## Рассмотренные варианты

### Трей

#### 1. `fyne.io/systray` (выбран)

Поддерживаемый форк `getlantern/systray`: только иконка + меню, без остального Fyne.

- ✅ +~1 МБ к бинарнику; на Windows — чистый WinAPI без cgo.
- ✅ Де-факто стандарт «просто трей» в Go-экосистеме.
- ❌ На macOS требует cgo (Objective-C) и запуска на главном потоке → кросс-компиляция darwin-бинарника с Windows невозможна, нужна сборка на macOS (локально или CI).
- ❌ Чтобы на macOS иконка не появлялась в доке, нужен `.app`-бандл с `LSUIElement` (task-10).

#### 2. Wails v3 (встроенные трей и окна)

- ✅ Трей и нативные окна из одной библиотеки.
- ❌ На момент решения — alpha; фиксировать на нём релиз рискованно.
- ❌ +8–12 МБ, cgo на обеих ОС, зависимость от WebView2-рантайма на Windows.

#### 3. Полноценный Fyne GUI

- ❌ +30–50 МБ, OpenGL-рендеринг, постоянная нагрузка на GPU — несоразмерно фоновому агенту.

### Форма настроек

#### 1. Встроенный локальный веб-интерфейс (выбран)

Агент поднимает HTTP-сервер строго на `127.0.0.1`; страница настроек (HTML/CSS/JS через `go:embed`, без фреймворков) открывается в браузере пользователя по клику в меню трея.

- ✅ +0.2–0.5 МБ к бинарнику: `net/http` и `go:embed` — stdlib.
- ✅ Одна вёрстка на обе ОС; можно сделать аккуратно и красиво.
- ✅ Проверенный паттерн для фоновых сервисов: Syncthing, qBittorrent WebUI.
- ❌ Настройки — вкладка браузера, а не нативное окно. Для редко открываемого экрана настроек приемлемо.
- ⚠️ Требует защиты: слушать только loopback + одноразовый auth-токен в URL (см. spec.md).

#### 2. Wails / webview-окно

- ❌ Те же минусы, что у Wails-трея: alpha либо (v2) отсутствие трея, cgo, WebView2.

#### 3. Только меню трея + «Открыть конфиг в редакторе»

- ✅ Почти бесплатно; оставляем как дополнительный пункт меню.
- ❌ Не отвечает требованию «удобная и красивая форма».

### Хранение токена

#### 1. Системное хранилище учётных данных (выбран)

`github.com/zalando/go-keyring`: Windows — Credential Manager (через `danieljoos/wincred`, без cgo), macOS — Keychain (через `/usr/bin/security`, без cgo).

- ✅ Токен шифруется ОС и привязан к пользователю; безопаснее переменной окружения, видимой всем процессам пользователя.
- ✅ Токен можно ввести в форме настроек — не нужно объяснять пользователю переменные среды.
- ✅ Библиотека без cgo на обеих целевых ОС.

#### 2. Переменная окружения `HA_TOKEN` (статус-кво)

- ❌ Неудобно задавать из GUI; токен виден в окружении любого процесса пользователя.
- Остаётся **резервным механизмом** для headless-запуска и обратной совместимости.

#### 3. Токен в YAML-конфиге

- ❌ Открытый текст на диске, риск попадания в бэкапы/репозиторий. Отвергнут ещё в v1.

## Решение

| Назначение | Выбор | Вес |
|---|---|---|
| Трей | `fyne.io/systray` | ~1 МБ |
| Настройки | локальный веб-UI: `net/http` + `go:embed` (stdlib) | <0.5 МБ |
| Токен | `github.com/zalando/go-keyring` | <0.5 МБ |

Итоговый бинарник — в пределах ~12 МБ, внешних рантаймов нет.

## Последствия

- **Сборка darwin** больше не кросс-компилируется с Windows (cgo systray): darwin-бинарник собирается на MacBook или в GitHub Actions (macos-раннер) — task-10.
- **Windows-бинарник** собирается с `-H=windowsgui` (без консольного окна) → stdout недоступен, логи переводятся в файл (spec.md, раздел «Логирование»).
- **macOS-дистрибутив** — `.app`-бандл с `LSUIElement=true`, чтобы агент жил только в строке меню.
- Появляется **hot-reload конфига**: настройки, сохранённые из веб-UI, применяются без перезапуска агента (пересоздание подписок).
- Приоритет источников токена: keyring → переменная окружения из `token_env`. Флаг `-no-tray` сохраняет прежний headless-режим.
- Если понадобится нативное окно настроек — точечная замена `internal/webui` на Wails v3 после его стабилизации, отдельным ADR.
