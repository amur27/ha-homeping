# Подготовка Home Assistant

Изменения на стороне Home Assistant минимальны: новые контейнеры и интеграции не нужны. Требуются только токены доступа.

## 1. Определить адрес Home Assistant

Агенту нужен стабильный адрес. Варианты (в порядке предпочтения):

1. **mDNS-имя** `homeassistant.local` — работает из коробки, если ОС клиентов поддерживает mDNS (macOS — всегда, Windows 11 — да).
2. **Статический IP / DHCP-резервирование** для Raspberry Pi в роутере, например `192.168.1.50` — надёжнее всего, рекомендуется настроить в любом случае.

Итоговый URL для конфига агента: `ws://<адрес>:8123/api/websocket`.

## 2. Создать long-lived access token

**Отдельный токен на каждое устройство** (Windows-ПК и MacBook) — чтобы отзыв одного не затрагивал другое.

1. Открыть веб-интерфейс HA → кликнуть по имени пользователя (левый нижний угол) → вкладка **Безопасность**.
2. Внизу раздел **Долгосрочные токены доступа** → **Создать токен**.
3. Имя: `notify-agent-windows` (и затем второй: `notify-agent-macbook`).
4. Скопировать токен сразу — он показывается один раз. Передать на целевую машину безопасным способом (не в мессенджер в открытом виде).

Отзыв токена — там же, в один клик.

## 3. Узнать entity_id датчиков

1. В HA: **Настройки → Устройства и службы → Сущности**.
2. Найти датчик (например, «Front Door»), открыть — поле **ID сущности**, например `binary_sensor.front_door`.
3. Выписать все нужные `entity_id` — они пойдут в конфиг агента.

## 4. Проверить доступность API (с любого компьютера в сети)

```bash
# Должен вернуть {"message": "API running."}
curl -H "Authorization: Bearer <ТОКЕН>" http://homeassistant.local:8123/api/

# Текущее состояние конкретного датчика
curl -H "Authorization: Bearer <ТОКЕН>" \
  http://homeassistant.local:8123/api/states/binary_sensor.front_door
```

Если ответ получен — HA готов, дальше см. [setup-clients.md](setup-clients.md).

## 5. Замечания по Docker

- Порт 8123 должен быть проброшен наружу контейнера (при `network_mode: host` — уже доступен). Если API отвечает из браузера — всё в порядке.
- В `configuration.yaml` ничего менять не нужно: WebSocket API включён всегда.
