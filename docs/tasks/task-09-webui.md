# Task 09 — Веб-интерфейс настроек

## Цель

Настройка агента без ручной правки YAML: страница в браузере (открывается из трея) для редактирования сущностей и параметров, ввода токена и проверки подключения. Сохранение применяется без перезапуска.

## Контекст

- Спецификация: [../spec.md](../spec.md), разделы 3.3, 8, 9, 10.
- Решение по стеку: [../adr/ADR-003-ui-stack.md](../adr/ADR-003-ui-stack.md) — stdlib `net/http` + `go:embed`, без внешних фреймворков и CDN.
- Зависимости: task-07 (Reload, атомарная запись, secrets), task-08 (пункт меню «Настройки…»).

## Шаги

1. Пакет `internal/webui`:
   - сервер строго на `127.0.0.1`, эфемерный порт; сессионный auth-токен (`crypto/rand`), передача через `?auth=` → cookie, иначе 403 (спека 8.1);
   - API из спеки 8.3; ошибки валидации `PUT /api/config` — 422 с русским описанием; токен HA нигде не отдаётся наружу.
2. Страница (`go:embed`, один HTML + CSS + ванильный JS): форма URL HA, параметры уведомлений и логирования, таблица сущностей (добавить/удалить/править, переключатель `states`/`template`), поле токена (password, только «задан/не задан»), блок статуса с кнопками «Проверить подключение» и «Тестовое уведомление». Аккуратный внешний вид: системный шрифт, тёмная/светлая тема по `prefers-color-scheme`.
3. Интеграция: `PUT /api/config` → атомарная запись YAML → `Reload()`; `POST /api/token` → keyring → `Reload()`.
4. Трей: активировать пункт «Настройки…» — запуск/переиспользование сервера и открытие браузера со свежей auth-ссылкой. При первом запуске без конфига/токена (спека, раздел 2) страница открывается автоматически.
5. Юнит-тесты: auth (403 без токена/с чужим токеном), `PUT /api/config` с невалидными данными, сериализация конфига в JSON и обратно в YAML.

## Критерии приёмки

- [ ] `go build ./...`, `go vet ./...`, `go test ./...` — чисто.
- [ ] «Настройки…» в трее открывает страницу в браузере по умолчанию; страница работает без интернета.
- [ ] Добавление сущности на странице → сохранение → уведомление по ней приходит **без перезапуска агента**; YAML на диске обновлён и валиден.
- [ ] Невалидные данные формы не проходят: понятная ошибка на русском, конфиг на диске не тронут.
- [ ] Токен, введённый на странице, оказывается в Credential Manager/Keychain; в YAML, логах и ответах API его нет; после сохранения агент подключается.
- [ ] Запрос с другого адреса интерфейса (не loopback) невозможен; запрос на loopback без auth-токена — 403.
- [ ] Первый запуск без конфига: агент стартует, статус «не настроен», страница настроек открылась сама.
