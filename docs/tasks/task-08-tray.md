# Task 08 — Трей

## Цель

Агент живёт в трее: иконка отражает статус супервизора, меню даёт управление (настройки, тест, пауза, reload, выход). Headless-режим сохраняется флагом `-no-tray`.

## Контекст

- Спецификация: [../spec.md](../spec.md), раздел 7 (иконка, меню), а также 4 (auth_invalid в трей-режиме) и 12 (коды выхода).
- Решение по стеку: [../adr/ADR-003-ui-stack.md](../adr/ADR-003-ui-stack.md) — `fyne.io/systray`.
- Зависимость: task-07 (статусы и методы супервизора).

## Шаги

1. Пакет `internal/tray`: инициализация `fyne.io/systray`, набор иконок из спеки 7.1 (`go:embed`, для Windows — `.ico`, для macOS — png-шаблон), подписка на статус супервизора, меню из спеки 7.2.
2. `cmd/agent`: `systray.Run` занимает главный поток (требование macOS); супервизор — в горутине. `-no-tray` полностью отключает трей и сохраняет прежний headless-запуск. Корректное завершение: «Выход» из меню и сигнал ОС ведут через один и тот же путь остановки.
3. Пункты «Настройки…» — до task-09 открывает страницу-заглушку не нужно: пункт скрыт или неактивен, включается в task-09. «Открыть конфиг» — системное открытие YAML (`explorer`/`open`).
4. Поведение при `auth_invalid`, отсутствии токена и невалидном конфиге — статусы и уведомления по спеке, без завершения процесса.
5. В `scripts/build.ps1` для Windows добавить `-H=windowsgui` (консольное окно больше не появляется; логи уже в файле после task-07).

## Критерии приёмки

- [ ] `go build ./...`, `go vet ./...`, `go test ./...` — чисто.
- [ ] На Windows 11: иконка в трее, статусы меняются (отключить Wi-Fi → «отключён», вернуть → «подключён»), консольного окна нет.
- [ ] «Тестовое уведомление», «Пауза уведомлений» (события в лог, но без уведомлений; чекбокс), «Перечитать конфиг», «Открыть конфиг», «Выход» (код 0) — работают.
- [ ] `auth_invalid` (испорченный токен) не завершает агент: статус «ошибка токена», одно уведомление.
- [ ] `-no-tray` даёт прежнее headless-поведение, включая коды выхода 2/3.
- [ ] Сборка darwin-бинарника выполняется на MacBook (cgo); иконка появляется в строке меню, меню работает. Упаковка в `.app` — task-10, на этом этапе допустима иконка и в доке.
