# Задачи реализации homeping

Каждая задача самодостаточна: содержит цель, контекст, шаги и критерии приёмки. Выполнять строго по порядку — каждая следующая опирается на результат предыдущей.

## Общие правила для всех задач

1. Перед началом прочитать [../spec.md](../spec.md) — спецификация первична.
2. Комментарии в коде — на русском; в начале каждого файла/пакета — комментарий с описанием его логики.
3. После каждой задачи: `go build ./...`, `go vet ./...`, `go test ./...` — чисто.
4. Задача закрыта, только когда выполнены все её «Критерии приёмки».

## Сквозной сценарий приёмки (после task-06)

1. Собрать бинарники для Windows и macOS.
2. Развернуть по [../setup-clients.md](../setup-clients.md) на обеих машинах.
3. Открыть дверь → на обеих машинах уведомление «🚪 Дверь открыта» в течение ~1 секунды.
4. Остановить контейнер HA на 1 минуту → уведомление о недоступности; запустить → уведомление о восстановлении, события снова приходят.
5. Усыпить/разбудить ноутбук → события возобновляются без ручных действий.
