# Task 01 — Каркас проекта

## Цель

Создать компилируемый скелет Go-проекта со структурой каталогов и обработкой флагов CLI по разделу 2 спецификации.

## Контекст

- Спецификация: [../spec.md](../spec.md), разделы 2 и 9.
- Репозиторий: корень `C:\Projects\HA-Notifications` (документация уже лежит в `docs/`).

## Шаги

1. `go mod init homecrier`, Go ≥ 1.22.
2. Создать структуру:
   ```
   cmd/agent/main.go      — точка входа
   internal/config/       — пока пустой пакет-заглушка с doc-комментарием
   internal/hass/         — то же
   internal/notify/       — то же
   scripts/               — пустой каталог (заполнится в task-06)
   ```
3. В `main.go`: разбор флагов `-config`, `-test`, `-version`; переменная `version = "dev"` (для подмены через ldflags в task-06); graceful shutdown по SIGINT/SIGTERM через `signal.NotifyContext`.
4. Функция определения пути конфига по умолчанию для текущей ОС (раздел 2 спецификации), с юнит-тестом.
5. `.gitignore`: бинарники, `dist/`, локальные конфиги с токенами.
6. `git init` и первый коммит (документация + каркас).

## Критерии приёмки

- [ ] `go build ./...`, `go vet ./...`, `go test ./...` проходят на Windows.
- [ ] `go run ./cmd/agent -version` печатает `dev` и завершается с кодом 0.
- [ ] Путь конфига по умолчанию корректен для Windows и macOS (проверено тестом с подменой ОС-специфичной части).
- [ ] В начале каждого файла — комментарий с описанием логики на русском.
