# Task 05 — Устойчивость: переподключение и уведомления о связи

## Цель

Обернуть `hass.Client.Run` в цикл переподключения с экспоненциальным бэкоффом и джиттером; реализовать уведомления «HA недоступен» / «связь восстановлена».

## Контекст

- Спецификация: [../spec.md](../spec.md), раздел 6 (переподключение), раздел 5 (уведомления об обрыве).
- Зависимости: task-03 и task-04 выполнены.

## Шаги

1. Пакет `internal/hass` (или отдельный супервизор в `cmd/agent`): цикл `for { Run(ctx); backoff.Wait(ctx) }`.
2. Бэкофф: 1 → 2 → 4 → … → максимум 60 секунд, джиттер ±20%; сброс в 1 секунду после успешной аутентификации. Вынести в маленький тестируемый тип `Backoff`.
3. `ErrAuthInvalid` НЕ ретраится — пробрасывается наверх, выход с кодом 3.
4. Отслеживание состояния связи: если соединение потеряно дольше 30 секунд и `on_disconnect: true` — одно уведомление «⚠️ Home Assistant недоступен»; при следующей успешной подписке — «✅ Связь с Home Assistant восстановлена». Повторных уведомлений о том же обрыве нет.
5. Завершение по ctx (SIGINT/SIGTERM) мгновенно прерывает и соединение, и ожидание бэкоффа; код выхода 0.

## Критерии приёмки

- [ ] Юнит-тесты `Backoff`: последовательность растёт до потолка 60 с, джиттер в пределах ±20%, сброс работает.
- [ ] Тест супервизора с фейковым клиентом: N обрывов подряд → N переподключений; auth_invalid → без ретрая.
- [ ] Ручная проверка: остановить контейнер HA на ~1 минуту → одно уведомление о недоступности; запустить → уведомление о восстановлении, события снова приходят без перезапуска агента.
- [ ] Ctrl+C завершает агента мгновенно с кодом 0 даже в середине ожидания бэкоффа.
